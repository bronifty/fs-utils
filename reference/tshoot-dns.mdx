# t-shoot dns

- dig the domain it will show you the ip address(es) and you can check if your dns rules have propagated

```sh
dig domain-name
```

### cloudfront specifics

- add a cname record for each alternate domain that you want routed to the cloudfront distribution
- route 53 has a redirect record to point from apex to www or vice versa. you only need one bucket and one distribution but multiple dns records
